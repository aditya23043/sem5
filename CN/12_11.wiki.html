1 = Important Terms =
 2 * Maximum Diameter = 232 bit time
 3     * Max distance between two nodes
 4 * Jam Signal = 32 - 48 bits
 5     * The moment any transmitting node experiences collision, it starts a jam signal which is a sequence of 10 10 10.... till 48 bits
 6 * Interframe Delay = 96 bit time
 7     * Delay before a node waits before transmitting (while the channel is idle)
 8     * Even the first transmission does not happan at t=0; it happens at t=96 bit time
 9     * If it receives any signal before this timeout happens, the node backs off
10 * Note:
11     * Round trip bits = 2 * 232 bit = 464 bit
12     * Ethernet frame size = 512 bits
13     * Subtract → 512 - 464 = 48 bits (jam signal)
14 * Jam signal goes in the end (in place of the CRC) and ensures that CRC check fails
15 
16 ```example
17 Eth frame size = 512
18 preamble = 8 bytes = 64 bits
19 total -> 576 bits
20 
21 If B starts sending at t=324, it will reach A at t=649 but A had already sent all the frame bits (576) and since not collision was detected in between, it won't retransmit
22 Hence we need to cap our propagation delay at 232 bits times
23 we can also increase the interframe delay to more than the propagation delay but that will impact idle delay by a LOT
24 ```

28 = Ethernet Switches =
29 * Earlier we used to use Hubs which used to just amplify the signal and transfer on all busses and for collisions we used CSMA/CD
30 * Now, we use switches in which no 2 nodes are sending on the same signal
31 * CSMA/CD not required but some people are still using hubs so CSMA/CD is still kept
32 * If nobody is sending, it gives full bandwidth (great)
33 * MUX logic used in ethernet switches
34 * Ethernet switches are plug-and-play, self-learning (not much overhead)
35     * Switches need not be configured
36 * Switches also have mapping tables (switch table) like routers but much simpler because eth frames contains the dest and src mac address
37 * Initially, it needs to do some broadcast for mapping ports to mac address
38 * a.k.a layer 2 switches
39 * routers are also called layer 3 switches
40 * We can have cascaded switches and it will still learn very fast

53 = Wireless Networks =
54 * TCP performs worse in the case of wireless technology due to the channel being even more unreliable can result in more congestion and that slows the transmission rate
55 * Requires a "base station"
56 * Those base stations are themselves connected by wire
57 * Note: they can be connected wirelessly but it would introduce some problems, so generally → wired
58 * Two types of wireless networks
59     1. Infrastructure : centralized controller
60     2. Ad-hoc : like p2p; theoretically good but practically complex
61         * No base stations
62         * Used for sensor deployment
63         * nodes can only transmit to other nodes within link coverage
64         * Eg: Zigbee, Bluetooth
65 
66 == Problems ==
67 * Multipath propagation in wireless link
68     * Receiver not just receives radio signals from Access point directly; but also signals bouncing off of the walls and other objects, etc
69 * Hidden Terminal problem
70 * Attenuation
71 
72 = CSMA/CA =
73 * Very difficult to detect collisions in wireless link
74 * So, we should avoid collisions
75 * Needs coordination
76 * Take permission from all nodes to transmit
77 * RTS message = Request-To-Send
78 * CTS message = Clear-To-Send
79 * send RTS. wait for CTS. if CTS comes, wait for some time and then start transmission
80 * much less efficient than wired but still a solution
81 * need to implement fairness
82 * Complexity: there could be many hidden terminals
83 
84 = CDMA =
85 * Code division multiple access
86 * each node gets a unique code
87 * encode the original message with the code
88 * Receiver decodes messages with key = node's code
89 * better than FDMA, TDMA
