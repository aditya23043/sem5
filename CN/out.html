19 = CCN =
20 
21 * currently we search for some content by searching the host which has that data
22     * we go to the machine and then that finds and returns the data
23     * pros: if you know the address, it is easy to find it there every time
24     * cons: if a lot of people want that data â†’ congestion
25     * challening when the content is important
26     * we want "content" but we are forced to go a device to find the content
27 * Soln: Content-Centric networking / Information Centric Network / Named Data Networking
28     * All are same thing just with different names
29     * Basically caching at routers
30     * earlier, we had caching only at the local level
31     * Start naming the content
32     * dont care about machine
33     * number of node visited is reduced -> faster
34     * more popular -> even one machine fails, we still have other machines
35 * Challenges
36     * Addressing
37         * How to access the "content" if we dont have
38     * Content ownership
39     * Updating content
40     * Authenticity of content
41         * Earlier, you were sure that that "machine" was trustable either by certificate or just trust
42     * DNS lookup challenges
43         * Number of Table entries increased significantly
44         * DNS size is still not much but the we are caching the whole content so we need to have optimised storage solution
45         * Need alternate DNS (traditional dns wont work)
46 * Now we need to implement security at the content level, not the machine level
47 * tries to combine ideas of P2P and CDN
48 * Unicast: one to one
49 * Multicast: one to many (spanning trees used, complex to implement)
50 * Broadcast: one to ALL (considered waste of resources, but simple to implement)
51 * Need optimisations at router storage like TTL or priority
52 > ICN conference
53 * Routers do pro-active caching even if that data is not requested yet but have a high change of being requested in the near future
54 * Prevents dDoS
55 * Side channel attack: fetch data through attack through which the targeted data is inferred
56     * Like population detected from overall power consumption in a local network
